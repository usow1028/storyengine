# Milestone v1.1: Draft Scale

**Status:** ✅ SHIPPED 2026-04-11
**Phases:** 9-12
**Total Plans:** 12

## Overview

StoryGraph v1.1 extended the shipped v1.0 consistency core from scene-level reviewed input to chapter-scale draft analysis. The milestone focused on draft containers, deterministic segmentation, incremental review resilience, scoped checking, revision diffing, and large-run inspection while preserving deterministic verdict truth, explicit provenance, and advisory-only soft priors.

## Milestone Summary

**Key Decisions:**

- Keep draft-scale ingestion and inspection contracts additive so legacy scene-scale paths continue to work.
- Fail closed on unresolved or stale approval state rather than silently widening scope or rewriting canonical state.
- Preserve verdict-kind triage as the fixed inspection frame even when grouping and filtering large runs.
- Keep soft-prior output advisory-only and separate from deterministic hard verdict truth across all v1.1 surfaces.

**Issues Resolved:**

- Chapter-scale drafts can now persist stable ordering, scope, and source references instead of collapsing into a single scene blob.
- Writers can retry selected segments, preserve prior good work, and reopen approval explicitly instead of corrupting canonical state.
- Check and diff flows now support explicit approved scope and deterministic comparison identity across runs and revisions.
- Large-run inspection no longer hides partial-state truth; stale, unresolved, and failed work is surfaced directly in API and UI outputs.

**Issues Deferred:**

- A fresh milestone audit artifact was not created before archival; the next milestone should reintroduce audit-first closure discipline.
- Heavy graph visualization, collaboration, and editor integration remain outside the shipped v1.1 scope.

**Technical Debt Incurred:**

- `STATE.md` formatting has drifted from some GSD CLI assumptions and should be normalized before the next archival cycle.
- Milestone ledger maintenance still needs manual cleanup when an existing `Active` block in `MILESTONES.md` overlaps with CLI-generated shipped entries.
