<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server config -> soft-prior loader | `snapshotDir`, `snapshotSet`, `genreWeights`, and `worldProfile` are trusted runtime dependencies, not request payload fields. |
| prior artifact JSON -> runtime evaluator | Snapshot artifacts are untrusted file contents until parsed by existing Zod schemas. |
| advisory evaluator -> hard verdict persistence | Soft-prior scores cross back into the runtime result only; they must not mutate hard verdict rows or run metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering / Information Disclosure | `SoftPriorRuntimeConfig.snapshotDir` | mitigate | Keep `snapshotDir` on `ExecuteVerdictRunInput.softPriorConfig` only; do not add any API request field or default `data/prior-snapshots` assumption. |
| T-07-02 | Tampering | `loadPriorSnapshotSet` and malformed prior artifacts | mitigate | Catch loader parse/JSON errors in `evaluateConfiguredSoftPrior` and return `status: "invalid_snapshot"` while preserving hard verdict success. |
| T-07-03 | Denial of Service | missing prior artifacts | mitigate | Convert `ENOENT` and absent snapshot source into `status: "missing_snapshot"` instead of throwing out of the hard check path. |
| T-07-04 | Tampering | hard verdict records and run metadata | mitigate | Invoke `evaluateConfiguredSoftPrior` only after `verdictRepository.saveMany(verdicts)` and add invariance tests for `verdictKind`, `findingId`, evidence fields, `triggerKind`, and `createdAt`. |
| T-07-05 | Information Disclosure | returned advisory object | mitigate | Return `assessment`, `rerankedRepairs`, and `repairPlausibilityAdjustments` only; never return raw `snapshotSet`, raw artifacts, or snapshot `sourceWorkIds`. |
</threat_model>
