<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser route -> UI state | A user-controlled path segment is parsed as a run ID by `src/ui/App.tsx`. |
| browser -> inspection API | The UI fetches `/api/inspection/runs/:runId` from the browser. |
| package/tooling -> execution environment | New browser build/test tools enter the local development workflow. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | `src/ui/App.tsx` route parsing | mitigate | Parse only the run ID from `/inspection/runs/:runId`; do not add run listing, sharing, auth bypass claims, or browser-supplied prior config. |
| T-08-02 | Tampering / XSS | React shell rendering | mitigate | Render all UI strings as React text nodes and block `dangerouslySetInnerHTML` / `innerHTML` by acceptance grep. |
| T-08-03 | Denial of Service | build/browser tooling | mitigate | Use direct Vite and Playwright dependencies with explicit scripts so verification is reproducible and does not depend on transitive packages or watch mode. |
| T-08-04 | Spoofing | initial UI copy | mitigate | Use approved inspection-console copy only; do not add marketing, generation, repair-application, or graph-product language that misstates Phase 8 scope. |
</threat_model>
