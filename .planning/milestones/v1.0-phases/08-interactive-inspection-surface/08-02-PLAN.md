<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| check runtime -> storage | Runtime repair and advisory data crosses into persisted `verdict_runs.inspection_snapshot`. |
| prior evaluator -> snapshot | Corpus-derived advisory output is whitelisted before storage. |
| storage JSON -> future API DTO | Persisted JSON will later be parsed before API exposure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Information Disclosure | `RunInspectionSnapshotSchema` | mitigate | Whitelist snapshot fields and regression-test absence of `sourceWorkIds`, `snapshotDir`, `snapshotSet`, raw `PriorSnapshot`, and raw corpus rows. |
| T-08-06 | Tampering | `executeVerdictRun` snapshot persistence | mitigate | Save hard verdict rows before advisory generation; compare enabled/disabled prior hard projections to prove advisory cannot mutate deterministic verdict truth. |
| T-08-07 | Repudiation | persisted inspection snapshot | mitigate | Store `runId` and `createdAt` in the snapshot and parse it through Zod on read so a future inspection response traces back to the checked run. |
| T-08-08 | Denial of Service | snapshot storage shape | mitigate | Persist only run-scoped repairs and advisory summaries, not raw corpus artifacts or full prior snapshots. |
</threat_model>
