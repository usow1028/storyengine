<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| run-selection request -> repository lookup | Explicit base selectors must resolve only to deterministic comparable runs. |
| diff engine -> finding-level payload | Summary arrays and itemized trace output must stay consistent with each other. |
| inspection payload -> API consumers | Diff items must stay traceable but lean, and must not leak advisory internals or full snapshots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02 | Integrity | explicit base selector | mitigate | Resolve `baseRevisionId` only through deterministic comparison identity and reject incompatible scopes. |
| T-11-03 | Repudiation | finding-level diff trace | mitigate | Emit itemized diff entries with canonical IDs and scope labels instead of only summary arrays. |
| T-11-04 | Information Disclosure | diff payload shape | mitigate | Keep diff items lean and separate advisory/inspection-only details from diff truth. |
| T-11-05 | Integrity | comparable-run selection | mitigate | Prefer explicit `baseRunId`, then deterministic comparable `baseRevisionId`, then `previousRunId`; never silently mix scopes. |
</threat_model>
