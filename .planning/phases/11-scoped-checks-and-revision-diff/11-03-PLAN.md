## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API request -> service selector | Public callers can request scoped execution and explicit comparison selectors. |
| service diff payload -> API response | Stable public DTOs must remain traceable, lean, and free of internal leakage. |
| phase gate -> user trust | The full suite must prove scoped checks and diff selectors work together, not only in isolation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | scoped-check API | mitigate | Keep `/check` fail-closed for unresolved scopes and preserve the full-session guard when `scopeId` is omitted. |
| T-11-03 | Repudiation | inspection API diff items | mitigate | Expose finding-level scope labels and canonical trace IDs through stable public schemas. |
| T-11-04 | Information Disclosure | API response serialization | mitigate | Keep diff items lean and avoid leaking full inspection snapshots or prior-runtime internals. |
| T-11-05 | Integrity | explicit base selector API | mitigate | Reject ambiguous or incompatible selectors with stable 409/404 JSON instead of silent fallback. |
</threat_model>
