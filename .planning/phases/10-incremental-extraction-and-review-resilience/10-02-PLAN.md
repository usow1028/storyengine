<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reviewed segment -> canonical promotion | Corrected candidate payloads and approval state decide whether canonical storage changes. |
| approved segment -> retry or patch path | Reopened segments must not silently remain approved after material change. |
| session snapshot -> manual /check gate | The manual check route must not trust stale or partially approved aggregate state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | approved segment mutation | mitigate | Clear approval in the same transaction as material edits or explicit retry reset, and require explicit reapproval before the segment contributes again. |
| T-10-03 | Integrity | aggregate session state | mitigate | Recalculate session state after every patch/approval change and serialize `stale` plus progress counts so reopened state stays visible. |
| T-10-05 | Tampering | full-session manual check | mitigate | Keep the existing full-session gate and add an explicit per-segment `approved && current` check before running `executeVerdictRun()`. |
</threat_model>
