| shell -> triage list | Grouped verdict rows expose provenance and warning context that users rely on before selection. |
| shell -> detail panel | Deeper provenance rendering must stay explicit and separate from deterministic/advisory verdict semantics. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-12-02 | Integrity | grouping/filter UI | mitigate | Keep verdict kind as the outer grouping order and derive section/chapter groups only within each bucket. |
| T-12-03 | Repudiation | row/detail provenance | mitigate | Render provenance from payload metadata backed by stored IDs and source summaries, not inferred copy. |
