<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| verdict run -> inspection snapshot | Run-time operational counts become durable inspection metadata and must stay sanitized. |
| provenance records + ingestion session snapshot -> inspection payload | Source labels and spans are projected from stored trace data into user-visible inspection metadata. |
| inspection payload -> public API response | Additive metadata must remain traceable and must not leak raw session internals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Integrity | operational summary | mitigate | Persist explicit warning counts with the run and expose them prominently in the inspection payload. |
| T-12-03 | Repudiation | provenance projection | mitigate | Resolve row/detail provenance from persisted provenance IDs and session segment metadata, not ad-hoc labels. |
| T-12-04 | Information Disclosure | inspection payload | mitigate | Serialize sanitized counts, labels, source refs, and source spans only; do not inline raw snapshots or prompt artifacts. |
</threat_model>
