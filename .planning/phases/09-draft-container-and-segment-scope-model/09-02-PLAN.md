<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| domain draft plan -> SQL | Zod-parsed draft metadata crosses into PostgreSQL/pg-mem. |
| legacy rows -> enriched snapshot | Old ingestion rows without draft columns are loaded into the new snapshot contract. |
| review boundary patch -> stored source ref | User-edited offsets update persisted source refs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `saveDraftPlan` and row parsing | mitigate | Parse `DraftSubmissionPlanSchema`, `DraftSectionSchema`, and `DraftCheckScopeSchema` before writing/returning rows. |
| T-09-02 | Repudiation | `source_text_ref` persistence and patch updates | mitigate | Store LF-normalized source refs as JSONB and update offsets during `applySegmentPatch`; verify source-slice round trips. |
| T-09-03 | Denial of Service / Tampering | `0005_draft_scope.sql` and legacy loading | mitigate | Use additive nullable columns, `CREATE TABLE IF NOT EXISTS`, and compatibility synthesis for old rows instead of destructive backfill. |
| T-09-04 | Tampering | API response compatibility | transfer | Plan 09-03 owns additive API schema/serialization compatibility tests. |
</threat_model>
