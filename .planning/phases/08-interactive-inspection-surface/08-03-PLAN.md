<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client -> Fastify route | Untrusted `runId` path params cross into repository lookups. |
| storage rows -> inspection DTO | Verdict rows, run rows, evidence JSON, and snapshot JSON are transformed into a browser-safe DTO. |
| stored snapshot -> API response | Persisted advisory and repair data become read API output only after Zod parsing and redaction tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-09 | Information Disclosure | `GET /api/inspection/runs/:runId` | mitigate | Validate params, return stable 404 for absence, keep Phase 8 local/internal, and add no run-listing or public sharing route. |
| T-08-10 | Information Disclosure | inspection DTO | mitigate | Build a DTO, not raw rows; response and service tests assert no `sourceWorkIds`, `snapshotDir`, `snapshotSet`, raw `PriorSnapshot`, stack traces, or server prior config. |
| T-08-11 | Spoofing / Tampering | hard verdict versus advisory fields | mitigate | Keep deterministic verdict fields, repair candidates, and advisory signal in separate DTO branches; advisory cannot change group counts or selected verdict kind. |
| T-08-12 | Denial of Service | run payload builder | mitigate | Keep API run-scoped; expose summaries and collapsed trace-ready arrays rather than raw corpus or prior artifacts. |
| T-08-13 | Repudiation | inspection traceability | mitigate | Include `runId`, `storyId`, `revisionId`, `previousRunId`, `triggerKind`, `createdAt`, `findingId`, `reasonCode`, event IDs, state boundary IDs, and rule version IDs. |
</threat_model>
