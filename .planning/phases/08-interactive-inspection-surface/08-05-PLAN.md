<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DTO evidence trace -> DOM | Structured IDs, summaries, and arrays become interactive browser elements. |
| advisory output -> user interpretation | Corpus-derived pattern signals could be mistaken for hard verdict truth. |
| long trace data -> browser layout | Large arrays cross into expandable UI state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-19 | Spoofing | `SoftPriorAdvisoryBand` | mitigate | Mandatory separate heading and boundary note; advisory labels never use verdict/correction/truth/rule authority language. |
| T-08-20 | Repudiation | `TraceFields` | mitigate | Expose finding ID, reason code, conflict path, missing premises, supporting findings, event IDs, state boundary IDs, and rule version IDs. |
| T-08-21 | Denial of Service | long trace arrays | mitigate | Collapse arrays after six rows and expand in place without changing layout width. |
| T-08-22 | Information Disclosure | advisory rendering | mitigate | Block raw prior fields in component acceptance checks and render only sanitized DTO fields. |
| T-08-23 | Tampering / XSS | timeline and trace text | mitigate | Render through React text nodes only and keep `dangerouslySetInnerHTML` / `innerHTML` absent. |
</threat_model>
